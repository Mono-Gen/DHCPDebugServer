# DHCPDebugServer マニュアル v1.1.1 (日本語)

## 概要
本ツールは、DHCPクライアントの実装テストやネットワークのトラブルシューティングを目的とした、デバッグ機能付きのDHCPサーバです。

## 使用前の準備
1. **管理者権限**: 本アプリは特権ポート（UDP 67）を使用するため、必ず `DHCPDebugServer.exe` を右クリックして「管理者として実行」を選択してください。
2. **ポート競合の確認**: Windowsの「インターネット接続共有 (ICS)」が有効な場合、既にポート67が使用されており起動できません。サービス設定から一時的に停止してください。

## 使い方

### 1. サーバの開始
- **ネットワークIF**: DHCPをリッスンするアダプタを選択します。リストに目的のIFがない場合は、右側の更新ボタン（↻）を押してください。
- **IPプール開始/終了**: クライアントに割り当てるIPアドレスの範囲を指定します。
- **サーバ開始**: ボタンを押すと待機状態になります。起動に失敗する場合は、管理者権限とポート67の空きを確認してください。

### 2. ダッシュボード
- **稼働状況**: サーバが正常に動作しているかを表示します。
- **リース一覧**: 現在IPを割り当てているMACアドレスと、その有効期限が表示されます。

### 3. 高度なデバッグ (Debug Tab)
クライアントの挙動をテストするための強力な機能が利用できます。

- **すべての応答をドロップ**: 有効にすると、サーバはパケットを受信しても一切返信しません（沈黙状態）。
- **再リース要求を無視**: クライアントがIP更新要求（REQUEST）のみを無視します。
- **強制的に NAK を返答**: クライアントの要求に対して「拒否」を返します。
- **応答遅延 (ms)**: サーバの返信をあえて遅らせ、クライアントのタイムアウト処理を確認できます。
- **MACフィルタリング**: 特定の端末からのパケットのみを無視リストに追加できます（改行区切り）。MACアドレスは `00:11:22...` または `00-11-22...` の両方の形式に対応しています。

### 4. 通信ログ
- すべてのDHCPメッセージ（DISCOVER, OFFER, REQUEST, ACK, NAK 等）がリアルタイムに表示されます。
- **表示フィルタ**: Wireshark風の論理演算子 (`&&`, `||`) が使用可能です。大文字小文字を区別せず、特定の文字列の有無で絞り込めます（例: `08:0d:d2 && ACK`）。
- ID（Transaction ID）を追うことで、どのシーケンスが進行中かを確認可能です。
- **注意 (OSの制限)**: Windows OSのネットワーク仕様上、通信ログには選択したインターフェース以外のインターフェースからのDHCPメッセージも表示される場合があります。

## 特徴と互換性機能
- **高い互換性**: 本サーバは常にブロードキャスト（255.255.255.255）およびブロードキャストフラグ（0x8000）を使用して返信します。これにより、ユニキャストACKを受け付けない一部の特殊な機器でも確実に接続が可能です。
- **ノイズフィルタリング**: 以下のパケットは自動的に無視（ログにも出さない）されます：
  - 選択したIFのサブネット外からのパケット。
  - 他のDHCPサーバ宛のパケット（Server Identifierが不一致）。

## トラブルシューティング
- **ログに何も出ない**: 
  - ネットワークIFの選択が正しいか、ケーブルが接続されているか確認してください。
  - WindowsファイアウォールでUDP 67/68の通信が許可されているか確認してください。
  - クライアントのIP設定が自動取得（DHCP）になっているか確認してください。
- **起動時にエラーが出る**: 他のプロセスの競合（ICSなど）か、管理者権限の不足が主な原因です。
